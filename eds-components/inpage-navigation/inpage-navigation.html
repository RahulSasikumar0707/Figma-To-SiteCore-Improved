Component: Inpage Navigation
  • 'eds-inpage-navigation' class is mandatory at the parent level
  • Sitecore should be able to add/remove classes on parent level eds-inpage-navigation div
  • Add class 'highlight-hash-location' to highlight the active section in navigation based on the scroll position.
  • Developer can choose to use either desktop or mobile navigation or both based on the requirement.
  • Sitecore should be able to add/remove classes on both nav-desktop and nav-mobile div based on the requirement.
  • Sitecore should be able to add/remove 'inpage-scroll-trigger' class on anchor tag based on the requirement. This class is used to trigger the scroll event and to identify the active section while scrolling.
  • By default, make sure to add first section name in the active-section-title div. This will be updated based on the scroll position to show the current active section name in mobile view while scrolling.
  • Sitecore should wrap each section of components inside a div with 'inpage-scroll-target' class and a unique id to identify the section.
  • The id will be used to scroll to the section when the corresponding navigation link is clicked. Make sure the id is unique and does not contain any special characters or spaces and should match the href value of the corresponding navigation link. For example, if the navigation link has href="#section1", then the corresponding section should have id="section1". This is important for the scroll functionality to work correctly.
  • Also, make sure to add 'data-name' attribute with the section name to update the active section name in mobile view while scrolling.
  • Sitecore should be able to add/remove classes on 'inpage-scroll-target' div based on the requirement.
